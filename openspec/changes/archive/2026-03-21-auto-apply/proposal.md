# 变更提案：俱乐部任务自动签到/签退漏洞检查

## 目标
通过 OpenSpec 为“自动签到/签退”建立可验证规范，并基于规范检查当前实现中的逻辑漏洞、时序问题、空指针风险和错误处理缺陷。

## 范围
- 代码范围：`app/src/main/java/com/example/autorun/helper/App.java`
- 功能范围：
    - `runSignInOrBack()`
    - `runSignInOrBackForWorker()`
    - `resolveActionByWindow()`
    - `inWindow()`
- 检查维度：
    - 时间窗口判定
    - 签到/签退状态机一致性
    - 空值与异常处理
    - 返回码映射与重试策略
    - UI线程调用与后台任务行为一致性

## 非目标
- 不修改跑步上报逻辑（`runRun()`）
- 不重构网络请求层（`Request` 类）
- 不新增服务端能力

## 风险
- 时间边界（如 08:00:00、09:00:00）可能引发重复或误判
- 后台任务与前台按钮触发逻辑不一致，造成结果分叉
- `signInTf` 字段语义依赖服务端，若假设错误会误签或漏签
- 错误码被压缩为 `RETRY`，可能掩盖真实失败原因

## 验收标准
1. 有完整的自动签到/签退规格文档（含状态转移与窗口规则）
2. 至少列出 8 条可执行测试场景（正常、边界、异常）
3. 输出漏洞清单（含严重级别、触发条件、影响、建议）
4. 每个漏洞都有对应修复建议与回归测试点
